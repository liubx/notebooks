# 平台开发备忘（2022）

## 项目管理信息结构

统一的概括性项目描述：项目性质（科研/展示/实际使用），定位对象/人员规模，室内定位面积
- 地图相关描述、CAD 文件
- 硬件产品选型、硬件厂商管理
- 客户关系（甲方/乙方/最终客户）
- 项目联系人信息
- 时间节点（谈判/启动/部署/验收/交付）
- 项目来源
- 功能需求（已有功能/定制开发）
- 项目阶段：前期沟通 / 合同沟通 / 签约 / 开发 / 部署 / 完成

## 平台架构

```
platform-app
platform-frontend

  - gateway

  - prod-config / dev-config
```


## SAAS 服务数据表

- 位置服务：t_position / t_fence / t_alarm
- 设备服务：t_anchor / t_camera
- 地图服务：t_space / t_floor / t_polygon / t_poi / t_scenegraph
- 项目配置：t_project (config, 需手动更新)

## 接口规范

1. 调用接口 URL 需可统一修改
2. 统一授权方式（用户/密码/项目）
3. 补全接口（按 swagger 为准）
4. 规范命名/描述/header/参数
5. 保证可输出标准接口文档
6. 接口可用自动化测试

接口访问方式：
- 管理接口：用户登录获取 token → Authorization
- 定位平台接口：apiKey/apiSecret 或 schema/Authorization

## 测试流程

1. 针对产品线的测试流程
2. 定位平台测试计划（功能/接口/性能）
3. 测试环境和数据需求
4. 最好接上正式设备测试
5. 禅道 BUG 管理流程规范

## 代码 Review 要点
