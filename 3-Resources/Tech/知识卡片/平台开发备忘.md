## 平台开发备忘（2022）

### 项目管理信息结构

- 功能需求（已有功能/定制开发）
- 项目阶段：前期沟通 / 合同沟通 / 签约 / 开发 / 部署 / 完成

### SAAS 服务数据表

- 位置服务：t_position / t_fence / t_alarm
- 设备服务：t_anchor / t_camera
- 地图服务：t_space / t_floor / t_polygon / t_poi / t_scenegraph
- 项目配置：t_project (config, 需手动更新)

### 接口规范

1. 调用接口 URL 需可统一修改
2. 统一授权方式（用户/密码/项目）
3. 补全接口（按 swagger 为准）
4. 规范命名/描述/header/参数
5. 保证可输出标准接口文档
6. 接口可用自动化测试

接口访问方式：
- 管理接口：用户登录获取 token → Authorization
- 定位平台接口：apiKey/apiSecret 或 schema/Authorization

### 测试流程

1. 针对产品线的测试流程
2. 定位平台测试计划（功能/接口/性能）
3. 测试环境和数据需求
4. 最好接上正式设备测试
5. 禅道 BUG 管理流程规范

## 后端修改建议（Notion 迁移）

### API 接口修改

- 围栏人数统计接口 `/fences/access` → `/fences/statistics/accesses`
- 添加围栏进出记录分页接口 `/fences/accesses`
- `trackableTypeIds` 统一改为 `typeIds`
- 添加围栏类型接口（KEEP_IN/KEEP_OUT/ACCESS_LIMIT/OVERTIME_STAY/OVERTIME_LEAVE/STATISTIC）
- 围栏返回增加 `typeId` 和 `type` 对象

### API 参数描述规范

| 参数 | 说明 |
|------|------|
| current / pageSize | 分页（缺省 1/10） |
| offset / limit | 偏移分页（缺省 0/10） |
| sort / order | 排序（ascend/descend，缺省 descend） |
| search | 搜索关键字 |
| spaceId / mapId / floor | 空间/地图/楼层 |
| online / bound / isActive | 状态筛选 |
| startTime / endTime | 时间范围 |
| ids / typeIds / fenceIds / trackableIds | ID 列表筛选 |

### 博世&南讯集成 API 对接

鉴权方式：
1. 登录获取 token → `X-AUTH-TOKEN` + `X-Schema` 头
2. 项目 `apiKey` + `apiSecret` 头

地图数据接口：
- 多边形：`/api/maps/{mapId}/polygons`
- 兴趣点：`/api/maps/{mapId}/pois`
- 模型：`/api/maps/{mapId}/scenegraphs`

位置数据获取：
1. WebSocket 实时推送：`/websocket/position/api/ws`
2. REST 查询：`/api/positions`（支持时间段筛选）
