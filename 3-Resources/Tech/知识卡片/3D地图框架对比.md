# 3D 地图框架对比

WebGL 3D 地图/模型框架选型对比。

## 可用性需求

1. 大模型加载后是否流畅
2. 手机上运行是否流畅
3. 加载动画模型
4. 加载点、线、面等图形
5. 加载热点图
6. 加载图标（带聚合）
7. 模型格式支持（gltf, glb, gltf 压缩）
8. 文档完善程度
9. 技术支持

## 框架对比

### harp.gl

- 大模型：略有卡顿，FPS 8-9，整体效果比 deck.gl 好
- 手机：地图流畅，demo 模型流畅
- 动画：支持 FBX、GLB 动画模型
- 图形：支持点线面
- 热点图：通过 geojson 实现
- 图标：支持，聚合需其他库
- 压缩模型：支持，FPS 25-30（Draco 压缩）
- 文档：一般，需自己找 class
- 技术支持：还可以，有 slack 交流

### Cesium

- 大模型：略有卡顿，FPS<10，与 three.js 基本一致
- 手机：地图流畅
- 动画：支持 GLB 动画
- 图形：支持
- 热点图：支持
- 模型：gltf/glb/压缩都支持，800mb 模型 FPS<10 但流畅
- 文档：完善
- 技术支持：一般，issue 回复不及时

### BabylonJS

- 适合中大型项目，尤其多媒体混杂或游戏/VR 项目
- 支持格式：glTF, OBJ, STL, .babylon
- 优点：功能全面灵活，模型不失真，有微软背景，详细 API 文档
- 缺点：学习难度大，大模型加载慢，100M 以上可能无法加载

### ThingJS

- 面向物联网 3D 可视化 PaaS 开发平台
- 适用：数据中心、仓储、学校、医院、安防、城市、园区、建筑等
- 优点：流畅，文档好
- 缺点：不开源免费（有免费套餐）

### Layabox

- 适用：大型游戏开发、广告、营销、教育、应用开发
- 支持格式：Spline
- 优点：裸跑性能堪比 APP，轻量易用，支持 AS3/TS/JS，开源免费
- 缺点：技术支持一般

### Filament (Google)

- 基于 C++ 的跨平台物理渲染引擎
- 用于 Google 地图和搜索
- 优点：文档极好，相当于 PBR 教材
- 缺点：核心 C++ 写的，暴露给 JS 的 API 很少，只支持 WebGL 2.0（iOS 浏览器不可用）

### twgl.js

- 基于 three.js 的 helper library
- 主要解决 WebGL API 繁琐问题
- 适合初学者，没有独特功能
