# Bug 严重程度分类

## 1. Blocker（阻塞）

阻碍开发或测试工作的问题。测试人员通常很少遇到。

## 2. Critical（严重）

系统无法执行、崩溃或严重资源不足、应用模块无法启动或异常退出、无法测试、造成系统不稳定。

具体包括：
- 严重花屏
- 内存泄漏
- 用户数据丢失或破坏
- 系统崩溃/死机/冻结
- 模块无法启动或异常退出
- 严重的数值计算错误
- 功能设计与需求严重不符
- 用户权限问题
- 安全问题
- 其它导致无法测试的错误

## 3. Major（主要）

影响系统功能或操作，主要功能存在严重缺陷，但不会影响到系统稳定性。

具体包括：
- 功能未实现
- 功能错误
- 系统刷新错误
- 语音或数据通讯错误
- 轻微的数值计算错误
- 系统所提供的功能或服务受明显的影响

## 4. Minor（次要）

界面、性能缺陷。

具体包括：
- 操作界面错误（包括数据窗口内列名定义、含义是否一致）
- 边界条件下错误
- 提示信息错误（包括未给出信息、信息提示错误等）
- 长时间操作无进度提示
- 系统未优化（性能问题）
- 光标跳转设置不好，鼠标（光标）定位错误

## 5. Trivial（轻微）

易用性及建议性问题。

具体包括：
- 界面格式等不规范
- 辅助说明描述不清楚
- 操作时未给用户提示
- 可输入区域和只读区域没有明显的区分标志
- 个别不影响产品理解的错别字
- 文字排列不整齐等一些小问题
- 建议
