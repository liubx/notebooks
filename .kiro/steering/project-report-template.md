# 项目任务汇报模版

当用户要求发送项目任务汇报到飞书群时，按以下模版执行。

## 前置条件

- 项目的 `0-总览.md` 中需要有飞书群组 `chat_id`（`oc_xxx`）
- 项目的 `1-任务.md` 中包含任务列表

## 发送规则

分两条消息发送：

### 第一条：任务进展

使用飞书 post 富文本格式，标题为 `📊 {项目名} 任务进展（{日期}）`

内容顺序：
1. **近两天已完成** — 只列最近两天内完成的任务（有 ✅ 且完成日期在近两天内）
2. **进行中任务** — 按分组列出所有未完成任务，每项 @ 负责人
3. **整体总结** — 总任务数、已完成数、进行中数，当前重点
4. **需要注意** — 逾期任务、高优先级问题、关联风险等

### 第二条：各负责人注意事项

标题为 `👤 {项目名} 各负责人注意事项`

按每位负责人分段，@ 后直接跟待办数和具体事项提醒：
- 逾期任务标 ⚠️ 提醒优先处理
- 需要协同的标注协同对象
- 高优先级任务重点提示

## @ 格式

飞书 post 消息中 @ 人使用 `{"tag":"at","user_id":"ou_xxx"}`，at 标签本身会显示名字，**不要在 at 后面再重复写名字**。

正确：
```json
{"tag":"at","user_id":"ou_xxx"},{"tag":"text","text":"（6项进行中）\n• ..."}
```

错误：
```json
{"tag":"at","user_id":"ou_xxx"},{"tag":"text","text":" 陆东（6项进行中）\n• ..."}
```

## 负责人 open_id 查询

通过 `lark-cli contact +search-user --query '姓名' --as user` 查询。

## 发送方式

```bash
lark-cli im +messages-send --chat-id oc_xxx --msg-type post --content '<json>' --as bot
```
