## 改进记录

### 2024-01-15: 技术学习笔记的同步规则

**ID**: `improvement-001`  
**类型**: 同步判断偏好  
**状态**: ✅ 启用

**问题描述**：
AI 询问"学习了 React Hooks"是否需要同步到飞书，但这类个人学习内容不应该同步。

**正确做法**：
- 所有标记为"学习"的技术笔记默认为私有内容
- 除非明确包含项目标签（#project/工作项目）或工作领域标签（#area/工作领域）
- 不再询问用户，直接标记为 #private

**适用场景**：
- 技术学习笔记
- 个人技能提升相关内容
- 在线课程学习记录

**示例**：
```
✅ "学习了 React Hooks" → 自动标记为私有
✅ "学习了 TypeScript 泛型" → 自动标记为私有
❓ "学习了 React Hooks 用于项目 A" → 询问用户（包含项目标签）
❓ "团队分享：React Hooks 最佳实践" → 询问用户（团队相关）
```

### 2024-01-16: 项目任务的默认负责人

**ID**: `improvement-002`  
**类型**: 任务管理偏好  
**状态**: ✅ 启用

**问题描述**：
创建项目任务时，如果没有指定负责人，AI 会询问或留空，但大部分任务都是我自己负责。

**正确做法**：
- 创建项目任务时，如果没有明确指定负责人
- 默认分配给用户本人（@我）
- 如果明确指定了负责人，则使用指定的负责人

**适用场景**：
- 创建项目任务
- 从日常笔记中提取任务

**示例**：
```
✅ "添加任务：完成设计文档" → 自动分配给 @我
✅ "添加任务：代码审查" → 自动分配给 @我
✅ "添加任务：完成设计文档，张三负责" → 分配给 @张三
```

### 2024-01-18: 知识卡片使用中文标题

**ID**: `improvement-004`  
**类型**: 笔记创建习惯  
**状态**: ✅ 启用

**问题描述**：
AI 创建知识卡片时，有时使用英文标题（如 "React-Hooks-Closure-Trap"），但我更喜欢中文标题。

**正确做法**：
- 知识卡片标题优先使用中文
- 格式：主题-具体内容（如 "React-Hooks-闭包陷阱"）
- 如果是纯英文概念，可以使用英文（如 "JWT-Authentication"）

**适用场景**：
- 创建技术知识卡片
- 从日常笔记提炼知识点

**示例**：
```
✅ "React-Hooks-闭包陷阱" （推荐）
✅ "Python-装饰器模式" （推荐）
✅ "JWT-Authentication" （纯英文概念可以用英文）
❌ "React-Hooks-Closure-Trap" （避免）
```

### 2024-01-19: 任务截止日期的默认值

**ID**: `improvement-005`  
**类型**: 任务管理偏好  
**状态**: ✅ 启用

**问题描述**：
创建任务时如果没有指定截止日期，AI 会询问或留空，但大部分任务都是本周内完成。

**正确做法**：
- 如果没有明确指定截止日期
- 默认设置为本周五
- 如果今天是周五或周末，则设置为下周五

**适用场景**：
- 创建工作任务
- 创建项目任务

**示例**：
```
今天是周一：
✅ "添加任务：完成设计文档" → 截止日期：本周五

今天是周五：
✅ "添加任务：代码审查" → 截止日期：下周五

明确指定：
✅ "添加任务：完成设计文档，明天前" → 截止日期：明天
```

### 2024-01-21: 知识卡片提炼的最小长度

**ID**: `improvement-007`  
**类型**: 知识提炼偏好  
**状态**: ✅ 启用

**问题描述**：
AI 有时会为很短的内容创建知识卡片，但这些内容不值得单独成卡。

**正确做法**：
- 只有超过 200 字的技术内容才提炼成知识卡片
- 或者包含代码示例的内容（不限长度）
- 短小的技巧和提示保留在日常笔记中即可

**适用场景**：
- 从日常笔记提炼知识卡片
- 用户主动要求提炼

**示例**：
```
✅ 300 字的 React Hooks 原理分析 → 创建知识卡片
✅ 100 字 + 代码示例 → 创建知识卡片
❌ 50 字的简短技巧 → 保留在日常笔记
❌ 一句话的提醒 → 保留在日常笔记
```

## 使用说明

### 如何添加改进记录

通过对话告诉 AI：

```
你: "以后所有技术学习笔记都不要问我是否同步，直接标记为私有"
AI: "明白了！我已经记录下来：
    - 技术学习笔记默认为私有内容
    - 不再询问是否同步
    以后遇到类似内容会自动标记为私有。"
```

### 如何查看改进记录

```
你: "查看我的改进记录"
AI: "你有 8 条改进记录：
    1. 技术学习笔记默认私有（2024-01-15）
    2. 项目任务默认负责人（2024-01-16）
    3. 会议记录自动同步（2024-01-17）
    ..."
```

### 如何修改改进记录

```
你: "修改第 2 条记录，改为默认分配给张三"
AI: "已更新：项目任务默认负责人改为张三。"
```

### 如何删除改进记录

```
你: "删除第 5 条记录"
AI: "已删除：任务截止日期的默认值。"
```

### 如何禁用改进记录

```
你: "暂时禁用第 3 条记录"
AI: "已禁用：会议记录自动同步。需要时可以重新启用。"
```

## 注意事项

1. **改进记录会自动应用**：AI 会在后续操作中自动参考这些记录
2. **可以随时修改**：如果发现记录不合适，随时可以修改或删除
3. **不会影响历史数据**：改进记录只影响未来的操作，不会修改已有笔记
4. **可以导出和分享**：改进记录是纯文本文件，可以备份或分享给团队
5. **越用越智能**：随着改进记录的积累，AI 会越来越了解你的习惯
