## 系统概述

### 什么是这个系统？

这是一个基于 Obsidian 的智能知识管理系统，采用 PARA + Zettelkasten 混合方法，通过与 AI 助手对话来管理你的笔记、任务和知识。

### 核心特点

- **对话驱动**：通过自然语言对话完成所有操作，无需手动创建文件
- **智能同步**：AI 自动判断哪些内容需要同步到飞书，不确定时会询问你
- **PARA 组织**：清晰的文件夹结构（Projects/Areas/Resources/Archives）
- **知识网络**：通过双向链接和知识卡片构建个人知识库
- **任务管理**：自动聚合和分类所有任务

### 系统架构

```
你 ←→ AI 助手 ←→ Obsidian 笔记
              ↓
           飞书平台（工作内容自动同步）
```

## 快速开始

### 前置条件

1. **安装 Obsidian**：从 [obsidian.md](https://obsidian.md) 下载并安装
2. **配置 AI 工具**：使用支持 Skills 和 MCP 的 AI 工具（如 Kiro、Claude Desktop）
3. **安装必要的 Skills**：
   - `obsidian-markdown`（GitHub）
   - `json-canvas`（GitHub）
4. **配置 lark-mcp**（如果需要飞书同步）：
   - 安装 lark-mcp 服务器
   - 配置飞书应用凭证

### 初始化 Vault

第一次使用时，与 AI 助手对话：

```
你: "初始化 Obsidian 知识管理系统"

AI: "好的，我会为你创建 PARA 文件夹结构。请告诉我 vault 的路径。"

你: "/Users/yourname/Documents/MyVault"

AI: "已创建以下文件夹结构：
- 0-Daily（日常笔记）
- 1-Projects（项目）
  - Work（工作项目）
  - Personal（个人项目）
- 2-Areas（领域）
  - Work（工作领域）
  - Life（生活领域）
- 3-Resources（资源）
  - Tech（技术知识库）
  - Life（生活知识库）
- 4-Archives（归档）
- Attachments（附件）
- Templates（模板）

系统已准备就绪！"
```

## 核心工作流程

### 工作流程 1：日常记录

#### 流程图

```mermaid
graph LR
    A[开始一天] --> B[创建日常笔记]
    B --> C[记录工作内容]
    C --> D[AI 判断是否同步]
    D -->|工作内容| E[自动同步到飞书]
    D -->|私有内容| F[标记为私有]
    D -->|不确定| G[询问用户]
    G --> E
    G --> F
```

#### 详细步骤

1. **创建日常笔记**
   ```
   你: "记录今天的工作"
   AI: 创建 2024-01-15.md，包含工作、学习、生活三个板块
   ```

2. **记录内容**
   ```
   你: "今天开了个会，讨论了新项目的技术方案，参与者有张三、李四"
   AI: 在日常笔记中添加会议记录，自动提取关键信息
   ```

3. **智能同步判断**
   ```
   AI: "这是工作相关内容，我会自动同步到飞书。"
   或
   AI: "这个内容是否需要同步到飞书？"
   ```

### 工作流程 4：飞书双向同步

#### 流程图

```mermaid
graph TB
    A[Obsidian 任务] -->|更新| B[AI 检测变更]
    B --> C[同步到飞书]
    D[飞书任务] -->|更新| E[用户查询]
    E --> F[AI 同步回 Obsidian]
    G[冲突] --> H[AI 检测冲突]
    H --> I[询问用户选择]
```

#### 详细步骤

1. **Obsidian → 飞书**
   ```
   你: "把'完成设计文档'这个任务标记为完成"
   AI: "已更新任务状态，正在同步到飞书..."
       "同步完成！飞书任务也已标记为完成。"
   ```

2. **飞书 → Obsidian**
   ```
   你: "检查飞书上的任务更新"
   AI: "我检查了一下，有 2 个任务在飞书上被更新了：
       1. '代码审查' 已标记为完成
       2. '技术方案评审' 的截止日期改为明天
       是否同步这些更新到 Obsidian？"
   你: "同步"
   AI: "已同步完成。"
   ```

3. **冲突处理**
   ```
   AI: "检测到同步冲突：任务'完成设计文档'在两端都被修改了。
       
       Obsidian 版本（14:30）：
       - 状态：进行中
       - 截止日期：2024-01-20
       
       飞书版本（14:35）：
       - 状态：已完成
       - 截止日期：2024-01-22
       
       请选择：
       1. 保留 Obsidian 版本
       2. 保留飞书版本
       3. 手动合并"
   
   你: "保留飞书版本"
   AI: "好的，已使用飞书版本更新 Obsidian。"
   ```

## 最佳实践

### 1. 日常笔记习惯

**推荐做法**：
- ✅ 每天开始工作时创建日常笔记
- ✅ 随时记录想法和进展，不要等到下班
- ✅ 使用标签关联项目和领域
- ✅ 每周五生成周回顾

**避免**：
- ❌ 不要在日常笔记中写长篇大论（应该提炼成知识卡片）
- ❌ 不要忘记添加标签（影响搜索和分类）

### 2. 项目管理

**推荐做法**：
- ✅ 为每个项目创建独立文件夹
- ✅ 使用项目标签关联日常记录
- ✅ 定期更新项目状态
- ✅ 项目完成后及时归档

**避免**：
- ❌ 不要把所有任务都放在日常笔记中
- ❌ 不要忘记同步飞书（如果是工作项目）

### 3. 知识管理

**推荐做法**：
- ✅ 知识卡片保持原子化（一个卡片一个概念）
- ✅ 使用描述性标题
- ✅ 建立双向链接形成知识网络
- ✅ 定期回顾和更新知识卡片

**避免**：
- ❌ 不要创建过大的知识卡片
- ❌ 不要使用编号命名（如"卡片001"）

### 4. 飞书同步

**推荐做法**：
- ✅ 让 AI 自动判断是否同步
- ✅ 明确标记私有内容
- ✅ 定期检查同步状态
- ✅ 及时处理同步冲突

**避免**：
- ❌ 不要手动修改同步元数据
- ❌ 不要在两端同时编辑同一内容

### 5. 标签使用

**推荐的标签体系**：

```
主题标签：
#技术、#工作、#生活、#学习

状态标签：
#进行中、#已完成、#待办

优先级标签：
#重要、#紧急

项目标签：
#project/项目名

领域标签：
#area/领域名

任务分类：
#task/personal、#task/work、#task/project/项目名

同步标签：
#sync/feishu、#synced/feishu、#private
```

## 常见问题

### Q1: AI 如何判断内容是否需要同步到飞书？

**A**: AI 会分析内容特征：

**自动同步（工作内容）**：
- 包含项目标签 `#project/工作项目名`
- 包含工作领域标签 `#area/工作领域`
- 包含工作任务标签 `#task/work`
- 会议记录、技术决策记录（ADR）

**自动排除（私有内容）**：
- 包含个人任务标签 `#task/personal`
- 包含生活领域标签 `#area/生活领域`
- 位于 `1-Projects/Personal` 路径下

**询问用户**：
- 无明确特征的技术笔记
- 混合工作和生活内容

### Q2: 如何处理同步冲突？

**A**: 当 Obsidian 和飞书两端都修改了同一内容时：

1. AI 会检测到冲突并通知你
2. 显示两端的修改内容和时间
3. 提供三个选项：
   - 保留 Obsidian 版本
   - 保留飞书版本
   - 手动合并（AI 会创建冲突副本）

**建议**：
- 尽量避免在两端同时编辑
- 定期同步，减少冲突概率

### Q3: 可以不使用飞书同步吗？

**A**: 可以！飞书同步是可选功能。

如果不需要飞书同步：
- 不配置 lark-mcp
- AI 不会询问同步相关问题
- 所有功能正常使用（笔记、任务、知识管理等）

### Q4: 如何查看所有任务？

**A**: 有两种方式：

1. **通过对话**：
   ```
   你: "今天有哪些任务？"
   你: "本周的任务列表"
   你: "项目 A 的所有任务"
   ```

2. **查看任务中心**：
   打开 `任务中心.md` 文件，自动聚合所有任务

### Q5: 知识卡片和日常笔记有什么区别？

**A**: 

**日常笔记**：
- 按时间组织
- 记录当天的所有内容
- 临时性、流水账式

**知识卡片**：
- 按主题组织
- 专注单一概念
- 永久性、可复用

**工作流**：
日常笔记 → 提炼 → 知识卡片 → 知识网络

### Q6: 如何归档已完成的项目？

**A**: 

```
你: "归档'电商系统重构'项目"

AI: "已将项目移动到：4-Archives/2024/电商系统重构/
    - 保留了所有链接和标签
    - 添加了归档日期
    - 项目内容仍然可以搜索和访问"
```

### Q7: 可以自定义模板吗？

**A**: 可以！

1. 在 `Templates/` 文件夹中创建或修改模板
2. 使用变量占位符（如 `{{date}}`, `{{title}}`）
3. AI 会自动使用你的自定义模板

### Q8: 如何备份数据？

**A**: 

Obsidian vault 就是普通的文件夹，包含：
- Markdown 文件（笔记内容）
- 附件文件（图片、PDF 等）
- 配置文件（JSON）

**备份方式**：
- 使用 Git 版本控制
- 云同步（iCloud、Dropbox 等）
- 定期复制整个文件夹

### Q9: 支持移动端吗？

**A**: 

- Obsidian 有官方移动端 App（iOS/Android）
- 可以查看和编辑笔记
- AI 对话功能取决于你使用的 AI 工具是否支持移动端

### Q10: 如何搜索笔记？

**A**: 

1. **通过 AI 对话**：
   ```
   你: "查找关于 React 的笔记"
   你: "最近一周的会议记录"
   ```

2. **使用 Obsidian 搜索**：
   - 全文搜索
   - 按标签筛选
   - 按文件夹筛选
