# 告警规则说明

## 告警类型总览

平台支持 9 种告警规则，按触发机制可分为三大类：

| 分类 | 告警类型 | 触发对象 | 核心判断依据 |
|------|---------|---------|-------------|
| **围栏类** | 违规进入 | 定位对象 | 进入禁止区域 |
| | 违规离开 | 定位对象 | 离开限定区域 |
| | 超额 | 围栏区域 | 区域内人数超过上限 |
| | 缺额 | 围栏区域 | 区域内人数低于下限 |
| **时间类** | 滞留超时 | 定位对象 | 在某区域停留超过阈值 |
| | 暂离超时 | 定位对象 | 离开指定区域超过阈值 |
| | 手推车停滞超时 | 定位对象 | 手推车静止超过阈值 |
| **状态类** | 低电量 | 定位标签 | 电量低于阈值 |
| | 聚集 | 定位对象群体 | 短时间内多人聚集 |

## 各告警规则详细说明

### 1. 违规进入

| 属性 | 说明 |
|------|------|
| 触发条件 | 定位对象进入设定的围栏区域 |
| 关联配置 | 围栏 + 作用范围（标签/分组过滤） |
| 告警生成 | 对象位置落入围栏多边形内时触发 |
| 告警关闭 | 对象离开围栏区域后自动关闭 |

### 2. 违规离开

| 属性 | 说明 |
|------|------|
| 触发条件 | 定位对象离开设定的围栏区域 |
| 关联配置 | 围栏 + 作用范围（标签/分组过滤） |
| 告警生成 | 对象位置离开围栏多边形时触发 |
| 告警关闭 | 对象回到围栏区域内后自动关闭 |

### 3. 超额

| 属性 | 说明 |
|------|------|
| 触发条件 | 围栏区域内定位对象数量超过设定上限 |
| 关联配置 | 围栏 + 人数上限阈值 |
| 告警生成 | 区域内对象计数 > 上限时触发 |
| 告警关闭 | 区域内对象计数 ≤ 上限时自动关闭 |

### 4. 缺额

| 属性 | 说明 |
|------|------|
| 触发条件 | 围栏区域内定位对象数量低于设定下限 |
| 关联配置 | 围栏 + 人数下限阈值 |
| 告警生成 | 区域内对象计数 < 下限时触发 |
| 告警关闭 | 区域内对象计数 ≥ 下限时自动关闭 |

### 5. 滞留超时

| 属性 | 说明 |
|------|------|
| 触发条件 | 定位对象在某围栏区域内停留时间超过阈值 |
| 关联配置 | 围栏 + 滞留时间阈值 |
| 告警生成 | 对象在围栏内持续时间 > 阈值时触发 |
| 告警关闭 | 对象离开围栏区域后自动关闭 |

### 6. 暂离超时

| 属性 | 说明 |
|------|------|
| 触发条件 | 定位对象离开指定围栏区域的时间超过阈值 |
| 关联配置 | 围栏 + 暂离时间阈值 |
| 告警生成 | 对象离开围栏持续时间 > 阈值时触发 |
| 告警关闭 | 对象回到围栏区域内后自动关闭 |

### 7. 低电量

| 属性 | 说明 |
|------|------|
| 触发条件 | 定位标签上报电量低于设定阈值 |
| 关联配置 | 电量阈值（百分比） |
| 告警生成 | 标签电量 < 阈值时触发 |
| 告警关闭 | 标签电量恢复到阈值以上 / 手动关闭 |

### 8. 聚集

| 属性 | 说明 |
|------|------|
| 触发条件 | 短时间内多个定位对象在小范围内聚集 |
| 关联配置 | 聚集半径 + 人数阈值 + 持续时间 |
| 告警生成 | 指定半径内对象数量 ≥ 阈值且持续时间达标时触发 |
| 告警关闭 | 聚集人数降至阈值以下后自动关闭 |

### 9. 手推车停滞超时

| 属性 | 说明 |
|------|------|
| 触发条件 | 手推车类定位对象在某位置静止超过阈值 |
| 关联配置 | 停滞时间阈值 |
| 告警生成 | 手推车位置无变化持续时间 > 阈值时触发 |
| 告警关闭 | 手推车开始移动后自动关闭 |

## 告警触发逻辑流程图

```mermaid
flowchart TD
    Start([实时位置数据上报]) --> Parse[解析位置/状态数据]
    Parse --> RuleCheck{遍历已启用的告警规则}

    RuleCheck --> FenceType{规则类型?}

    %% ===== 围栏类告警 =====
    FenceType -->|围栏类| FenceCalc[计算对象与围栏的<br>位置关系]
    FenceCalc --> WhichFence{具体规则?}

    WhichFence -->|违规进入| InCheck{对象是否<br>在围栏内?}
    InCheck -->|是| InScope1{在作用范围内?}
    InScope1 -->|是| InAlarm[/生成 违规进入 告警/]
    InScope1 -->|否| Skip1([跳过])
    InCheck -->|否| InExist{已有告警?}
    InExist -->|是| InClose[/关闭 违规进入 告警/]
    InExist -->|否| Skip1

    WhichFence -->|违规离开| OutCheck{对象是否<br>在围栏外?}
    OutCheck -->|是| OutScope{在作用范围内?}
    OutScope -->|是| OutAlarm[/生成 违规离开 告警/]
    OutScope -->|否| Skip2([跳过])
    OutCheck -->|否| OutExist{已有告警?}
    OutExist -->|是| OutClose[/关闭 违规离开 告警/]
    OutExist -->|否| Skip2

    WhichFence -->|超额| CountUp[统计围栏内对象数量]
    CountUp --> OverCheck{数量 > 上限?}
    OverCheck -->|是| OverAlarm[/生成 超额 告警/]
    OverCheck -->|否| OverExist{已有告警?}
    OverExist -->|是| OverClose[/关闭 超额 告警/]
    OverExist -->|否| Skip3([跳过])

    WhichFence -->|缺额| CountDown[统计围栏内对象数量]
    CountDown --> UnderCheck{数量 < 下限?}
    UnderCheck -->|是| UnderAlarm[/生成 缺额 告警/]
    UnderCheck -->|否| UnderExist{已有告警?}
    UnderExist -->|是| UnderClose[/关闭 缺额 告警/]
    UnderExist -->|否| Skip4([跳过])

    %% ===== 时间类告警 =====
    FenceType -->|时间类| TimeCalc[计算持续时间]
    TimeCalc --> WhichTime{具体规则?}

    WhichTime -->|滞留超时| StayCheck{在围栏内<br>持续时间 > 阈值?}
    StayCheck -->|是| StayAlarm[/生成 滞留超时 告警/]
    StayCheck -->|否| StayExist{已离开围栏<br>且有告警?}
    StayExist -->|是| StayClose[/关闭 滞留超时 告警/]
    StayExist -->|否| Skip5([跳过])

    WhichTime -->|暂离超时| LeaveCheck{离开围栏<br>持续时间 > 阈值?}
    LeaveCheck -->|是| LeaveAlarm[/生成 暂离超时 告警/]
    LeaveCheck -->|否| LeaveExist{已回到围栏<br>且有告警?}
    LeaveExist -->|是| LeaveClose[/关闭 暂离超时 告警/]
    LeaveExist -->|否| Skip6([跳过])

    WhichTime -->|手推车停滞超时| CartCheck{静止时间<br>> 阈值?}
    CartCheck -->|是| CartAlarm[/生成 手推车停滞超时 告警/]
    CartCheck -->|否| CartExist{已移动<br>且有告警?}
    CartExist -->|是| CartClose[/关闭 手推车停滞超时 告警/]
    CartExist -->|否| Skip7([跳过])

    %% ===== 状态类告警 =====
    FenceType -->|状态类| StatusCalc[读取设备状态]
    StatusCalc --> WhichStatus{具体规则?}

    WhichStatus -->|低电量| BattCheck{电量 < 阈值?}
    BattCheck -->|是| BattAlarm[/生成 低电量 告警/]
    BattCheck -->|否| BattExist{已有告警?}
    BattExist -->|是| BattClose[/关闭 低电量 告警/]
    BattExist -->|否| Skip8([跳过])

    WhichStatus -->|聚集| ClusterCalc[计算指定半径内<br>对象数量]
    ClusterCalc --> ClusterCheck{数量 ≥ 阈值<br>且持续时间达标?}
    ClusterCheck -->|是| ClusterAlarm[/生成 聚集 告警/]
    ClusterCheck -->|否| ClusterExist{已有告警?}
    ClusterExist -->|是| ClusterClose[/关闭 聚集 告警/]
    ClusterExist -->|否| Skip9([跳过])

    %% ===== 告警处理 =====
    InAlarm & OutAlarm & OverAlarm & UnderAlarm --> Notify
    StayAlarm & LeaveAlarm & CartAlarm --> Notify
    BattAlarm & ClusterAlarm --> Notify

    Notify[推送告警通知<br>写入告警记录] --> End([等待下一次数据上报])

    InClose & OutClose & OverClose & UnderClose --> CloseLog
    StayClose & LeaveClose & CartClose --> CloseLog
    BattClose & ClusterClose --> CloseLog

    CloseLog[关闭告警<br>写入历史记录] --> End

    style ClusterAlarm fill:#ff6b6b,color:#fff
```

## 告警生命周期

```mermaid
stateDiagram-v2
    [*] --> 监测中: 规则启用
    监测中 --> 告警触发: 满足触发条件
    告警触发 --> 当前告警: 写入告警记录
    当前告警 --> 当前告警: 持续满足条件<br>（不重复生成）
    当前告警 --> 历史告警: 条件消除/手动关闭
    历史告警 --> [*]
    监测中 --> [*]: 规则禁用
```

## 关键设计要点

1. **去重机制**：同一对象对同一规则，在告警未关闭前不重复生成新告警
2. **实时位置驱动**：围栏类告警必须基于实时位置判断，不可根据历史位置生成
3. **自动关闭**：大部分告警在条件消除后自动关闭，低电量可手动关闭
4. **作用范围过滤**：围栏类告警支持按标签/分组过滤，只对指定对象生效
5. **告警排队**：大量告警同时触发时需排队处理，避免阻塞系统
