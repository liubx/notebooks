## ESXi 虚拟化

ESXi 服务器有两块物理网卡，分别对应两个 vSwitch。

### vSwitch0（WAN 侧，vmnic0 → 光猫）

```
物理网卡：vmnic0（1000 Mbps, Full）
│
├── IPTV Network（VLAN 45）
│   └── iptv ─── 00:0c:29:44:3b:bc
│
├── WAN Network（VLAN 0）
│   ├── router ─── 00:0c:29:e4:08:24
│   └── proxy ─── 00:0c:29:14:b4:e6
│
└── Wan Management Network（VLAN 0）
    └── vmk0: 192.168.10.2（ESXi 管理口）
```

### vSwitch1（LAN 侧，vmnic1 → 内网）

```
物理网卡：vmnic1（1000 Mbps, Full）
│
├── LAN Network（VLAN 0）
│   ├── iptv ─── 00:0c:29:44:3b:b2
│   ├── windows ─── 00:0c:29:ab:ec:09, 00:0c:29:ab:ec:ff
│   ├── router ─── 00:0c:29:e4:08:1a
│   ├── nas ─── 00:0c:29:fb:69:d3
│   └── proxy ─── 00:0c:29:14:b4:dc
│
└── LAN Management Network（VLAN 0）
    └── vmk1: 192.168.80.2（ESXi 管理口）
```

## Proxy 双网卡策略路由

Proxy 走了 Surge 网关（192.168.80.8），但代理服务的出站流量需要绕过 Surge 直接走光猫出去，否则会形成回环导致连接不上。通过 systemd-networkd 的策略路由解决。

### 配置文件

`/etc/systemd/network/10-eth0.network`（内网侧，192.168.80.5）：

```ini
[Match]
Name=eth0

[Network]
DHCP=yes
IPv6AcceptRA=no

[DHCPv4]
SendRelease=no
RouteMetric=512
```

`/etc/systemd/network/20-eth1.network`（光猫侧，192.168.10.5 + 策略路由）：

```ini
[Match]
Name=eth1

[Network]
DHCP=yes
IPv6AcceptRA=no

[DHCPv4]
SendRelease=no
RouteMetric=1024

[Route]
Table=100
Gateway=192.168.10.1

[RoutingPolicyRule]
SourcePort=6334
Table=100

[RoutingPolicyRule]
SourcePort=8888
Table=100

[RoutingPolicyRule]
SourcePort=8999
Table=100
```

### 路由逻辑

- eth0（内网 192.168.80.5）RouteMetric=512，eth1（光猫 192.168.10.5）RouteMetric=1024
- Proxy 走 Surge 网关，普通上网流量经 Surge 处理
- 代理服务端口（Snell 8888、Snell TLS 6334、V2Ray 8999）的出站流量通过策略路由表 100 强制走光猫网关 192.168.10.1，绕过 Surge，避免回环

### 防火墙（Photon OS iptables）

Proxy 运行 Photon OS，使用 iptables 管理防火墙。

配置文件：`/etc/systemd/scripts/ip4save`

```
-A INPUT -p tcp -m tcp --dport 22 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 6334 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 8888 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 8999 -j ACCEPT
-A INPUT -p tcp -m tcp --dport 5140 -j ACCEPT
```

修改后重启生效：

```bash
systemctl restart iptables
```

### NAT 回环问题

在家里通过公网 IP / 域名连接代理会失败（光猫不支持 NAT 回环），只能通过内网地址 192.168.10.5 连接。

临时解决方案：在内网 DNS（或 hosts）中将 home.liubx1988.com 解析到 192.168.10.5，这样在家里也能用域名连接。

## 出租屋网络

Mac Mini 部署在出租屋，运行 Surge 增强模式，完全接管本地网络（网关 + DHCP + DNS）。

| 项目 | 值 |
|------|------|
| 网段 | 192.168.88.0/24 |
| Mac Mini IP | 192.168.88.8 |
| Surge 模式 | 增强模式，接管所有流量 + DHCP 分配 |

出租屋所有设备的网关和 DNS 均由 Mac Mini 上的 Surge 分配和接管。

## Surge 配置

配置文件通过 iCloud 同步，路径：`~/Library/Mobile Documents/com~apple~CloudDocs/Applications/Surge/`

### 配置文件结构

| 配置文件 | 用途 | Ponte Server | 说明 |
|---------|------|-------------|------|
| Home.conf | 家里使用 | GZ-HOME-SNELL | 排除 192.168.10.0/24 和 192.168.80.0/24 |
| Default.conf | 出租屋 / 外面使用 | GZ-HOME-SNELL | 不排除家庭网段（通过代理回家） |
| Office.conf | 办公室使用 | GZ-OFFICE-SNELL | 排除 192.168.50.0/24（办公室网段） |

三个配置文件共享相同的 CA 证书和基础设置，通过 `#!include` 引入不同的 dconf 模块组合。

### dconf 模块文件

| 文件 | 内容 | 说明 |
|------|------|------|
| Home.dconf | 家庭代理节点 + 规则 + Host | 连接家里 Proxy 的节点定义 |
| HomeSpec.dconf | 家庭网段直连规则 | Home.conf 专用，192.168.10/80 网段 DIRECT |
| Office.dconf | 办公室代理节点 + 规则 + Host | 连接办公室 Proxy 的节点定义（托管配置） |
| OfficeSpec.dconf | 办公室网段直连规则 | Office.conf 专用，192.168.50 网段 DIRECT |
| Device.dconf | Surge Ponte 设备路由规则 | 通过 Ponte 隧道访问各设备 |
| Extra.dconf | 海外代理规则 | Microsoft/Kiro 等走海外节点 |
| Aboard.conf | Apple 直连 + 分流规则 | 大量域名规则，GEOIP CN 直连，其余漏网走代理 |

### 配置文件引入关系

```
Home.conf:
  Proxy/Group: HomeSpec + Home + Office + Extra + Aboard
  Rule:        HomeSpec + Home + Office + Device + Extra + Aboard
  Host:        HomeSpec + Home + Office + Device + Extra

Default.conf:
  Proxy/Group: Home + Office + Extra + Aboard
  Rule:        Home + Office + Device + Extra + Aboard
  Host:        Home + Office + Device + Extra

Office.conf:
  Proxy/Group: OfficeSpec + Office + Home + Extra + Aboard
  Rule:        OfficeSpec + Office + Home + Device + Extra + Aboard
  Host:        OfficeSpec + Office + Home + Device + Extra
```

注意：Home.conf 引入了 HomeSpec（家庭网段直连），Default.conf 没有引入 HomeSpec（家庭网段走代理回家）。

### Surge 规则概览

规则按优先级从高到低匹配：

1. **HomeSpec / OfficeSpec**：本地网段直连（仅在对应配置中生效）
2. **Home.dconf 规则**：
   - 192.168.10.0/24 和 192.168.80.0/24 → GZ-HOME-PROXY（外面时走代理回家）
   - 10.120.0.0/16 → GZ-HOME-VPN（VPN 网段走 Windows SOCKS5）
   - *.liubx1988.com → GZ-HOME-PROXY
   - home.liubx1988.com → DIRECT（避免回环）
3. **Office.dconf 规则**：
   - 192.168.50.0/24 → SZ-OFFICE-PROXY（外面时走代理到办公室）
   - 10.120.0.0/16 → GZ-OFFICE-PROXY
   - *.reliablesense.net → SZ-OFFICE-PROXY
4. **Device.dconf 规则**：各网段通过 Ponte 路由
5. **Extra.dconf 规则**：Microsoft / Kiro 等走海外代理（ABOARD-PROXY）
6. **Aboard.conf 规则**：Apple 域名直连，GEOIP CN 直连，其余走代理
