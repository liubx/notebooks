# map-mobile AI 代码阅读指南

> 本文档为 AI 辅助开发生成，用于快速理解项目结构、定位文件、修改代码。

---

## 一、项目概览

- **框架**: UmiJS (Alita) + React + TypeScript
- **类型**: 室内地图移动端 H5 应用
- **地图引擎**: `rsmap` SDK（自研室内地图渲染库）
- **UI 库**: `@alitajs/antd-mobile`
- **状态管理**: UmiJS useModel（每个 model 文件导出一个自定义 hook）
- **实时通信**: STOMP over WebSocket（`@stomp/stompjs`）
- **构建配置**: `config/config.ts`，appType=h5，publicPath=/map
- **路由模式**: browser history

### 核心功能
- 室内地图渲染（多楼层、3D/2D、卫星图、暗黑模式）
- 实时定位追踪（定位对象、标签、基站）
- 室内导航（路线规划、语音播报、偏航重路由、AR导航）
- 电子围栏 & 区域管理
- 告警通知
- 热力图 & 历史轨迹
- 搜索（室内POI + 腾讯地图公共地点）

## 四、Hooks 层

### 核心 Hooks（`src/hooks/`）

| Hook | 用途 | 关键点 |
|------|------|--------|
| index.ts | 统一导出入口 | **注意**: `useState` 实际是 `useSafeState`，`useEffect` 实际是 `useDeepCompareEffect` |
| useParam | URL参数双向同步 | 支持 number/boolean/string[]/Moment 类型，自动注册 window.setXxx |
| useWebsocket | 通用WebSocket数据订阅 | HTTP初始化 + STOMP实时推送，消息缓存批量合并(100ms) |
| useCommand | 远程命令执行 | STOMP订阅，eval执行服务端下发命令 |
| usePermission | 设备传感器权限 | iOS需requestPermission，Android默认启用 |
| useInitConfig | 初始化config model | 合并项目配置+props，增量更新 |
| useInitMap | 初始化map model | 合并项目配置+props，管理ref/selectId/followId |
| useSpeech | 语音播报 | 优先Android TTS → Howler音频精灵 → Web Speech API |
| useWindowState | 状态暴露到window | 自动注册 window.set/getXxx 方法 |
| useRequest | 增强版useRequest | 增加 debounceLoading 和 formatResult |
| useDirection | 设备方向传感器 | 区分Android/iOS事件 |
| useMovement | 设备运动检测 | 加速度阈值判断 |
| useLocation | 当前用户定位整合 | 整合位置+方向+运动状态，节流100ms |
| useInterval | RAF定时器 | 基于requestAnimationFrame的高精度定时 |
| useReferredState | state+ref同步 | 解决闭包旧值问题 |

## 九、关键数据流

### 1. 应用初始化
```
app.ts getInitialState()
  → URL获取 appId/appSecret
  → getApplication() 获取项目配置
  → 可选获取 trackable 绑定信息
  → 注入请求拦截器(apiKey/apiSecret头部)
```

### 2. 地图初始化
```
pages/index → RSMap/index.tsx
  → useInitConfig() 合并配置 → config model
  → useInitMap() 合并配置 → map model
  → rsmap SDK 渲染地图
  → 根据 show* 开关加载各 modules/*
```

### 3. 实时定位
```
trackable model
  → useWebsocket(getLatestPositions, '/topic/position/')
  → STOMP消息 → 缓存 → 100ms批量合并 → data更新
  → RSMap/modules/Trackables 渲染 Marker
```

### 4. 导航流程
```
用户选择终点 → PanelPlace "导航"
  → route model 设置起终点 → getRoute() 请求路线
  → PanelRoute 显示路线 → "开始导航"
  → navigation model 启动 Navigation 引擎
  → 实时位置匹配 → PanelManeuver 显示转向
  → 偏航 → PanelReroute → 重新规划
  → 到达 → PanelNavigation 显示完成
```

### 5. 配置控制（外部调用）
```
URL参数: ?showTrackables=true&enableSearchController=false
window方法: window.setShowTrackables(true)
  → useParam/useWindowState 同步到 React state
  → config/map model 更新
  → 组件响应式渲染
```

## 十一、注意事项

1. **hooks/index.ts 重命名**: `useState` 实际是 `useSafeState`（防卸载后setState），`useEffect` 实际是 `useDeepCompareEffect`（深比较依赖）。从 `@/hooks` 导入时注意这一点。
2. **多实例模式**: config 和 map model 支持通过 key 管理多个实例，默认 key 为 `'global'`。
3. **Zone 复用 Fence API**: zone service 实际调用的是 fence 的 API（`/fences`），通过 `types=STATISTIC` 过滤。
4. **坐标编码**: route service 中楼层号 ≥100 时使用位运算编码处理多建筑场景。
5. **useCommand 安全风险**: 使用 `eval()` 执行服务端下发命令，需确保服务端安全。
6. **useShelves 类型标注错误**: 返回类型标注为 `AlarmData[]`，实际应为 `ShelfData[]`。
