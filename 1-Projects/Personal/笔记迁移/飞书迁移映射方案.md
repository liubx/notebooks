| 类型 | 迁移方式 | 本地格式 |

| mindnote | 手动导出 FreeMind | .mm |
| slides | 手动导出 | .pptx |
| file（小文件） | `drive +download` | 原始格式 |
| file（大文件） | 跳过，保留飞书链接 | README.md 中标注 |

### 笔记性质分类原则

本地 `3-Resources/Tech/` 按笔记的"行动意图"分类，不按技术领域分：

| 目录 | 用途 | 打开它是为了 |
|------|------|------------|
| `代码片段/` | 可直接复制使用的命令、配置、代码 | 拿来用 |
| `知识卡片/` | 概念解释、技术对比、学习笔记 | 理解 |
| `技术决策/` | 选型原因、架构决策、权衡记录 | 回溯决策 |
| `问题解决/` | 问题描述、排查过程、解决方案 | 下次遇到快速定位 |
| `设计方案/` | 系统设计、架构图、方案文档 | 看架构（特指我们自己系统的设计） |
| `环境配置/` | 集群参数、服务器地址、登录信息 | 查地址、查参数 |

> 设计方案 vs 知识卡片的边界：设计方案放"我们自己系统的架构和设计"（系统架构图、概要设计说明书、软件项目资料），知识卡片放"通用技术知识"（3D地图框架对比、Bug等级分类标准）。

标签体系：
- 技术领域：`tech/xxx` 格式（如 `tech/k8s`、`tech/postgres`、`tech/gis`）
- 工作/个人：`work` 或 `personal`，用于区分来源。同步飞书时只同步 `work` 标签的内容，`personal` 留本地

> Tech 目录混合存放工作和个人笔记，不按来源分目录。同一个技术知识不管是工作用到的还是自己学的，按笔记性质归类。区分工作/个人通过标签实现。
