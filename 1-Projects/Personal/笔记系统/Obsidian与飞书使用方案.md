# Obsidian 与飞书使用方案

## 定位分工

| 维度 | Obsidian | 飞书 |
|------|---------|------|
| 定位 | 个人知识管理中心 | 团队协作平台 |
| 笔记 | 所有笔记的主库（PARA） | 团队共享文档（知识库） |
| 任务 | 本地视图 + 日记看板 | 权威数据源 + 团队协同 |
| 文档 | Markdown 长期存储 | 在线协作编辑 |
| 同步方向 | 飞书 → Obsidian（拉取）+ Obsidian → 飞书（推送） | — |

核心原则：**飞书负责团队协作，Obsidian 负责个人沉淀**。两者通过任务同步和文档迁移保持一致。

---

## 一、笔记管理

### 什么笔记放 Obsidian

所有笔记都以 Obsidian 为主库，按 PARA 结构组织：

| 类型 | 位置 | 示例 |
|------|------|------|
| 项目文档 | `1-Projects/` | 广州机场技术方案、上港业务流程 |
| 职责领域 | `2-Areas/` | 服务器运维记录、年度规划 |
| 参考资料 | `3-Resources/` | 技术知识卡片、行业解决方案 |
| 归档 | `4-Archives/` | 已完结项目、迁移笔记 |
| 日记 | `0-Daily/` | 每日记录、会议笔记 |

### 什么文档放飞书

飞书只放需要团队协作的内容：

| 类型 | 飞书位置 | 说明 |
|------|---------|------|
| 项目文档（团队共享） | 知识库：进行中项目 | 需要同事查看/编辑的文档 |
| 技术文档（团队共享） | 知识库：技术文档 | 运维手册、部署文档等团队参考 |
| 业务资料（团队共享） | 知识库：业务资料 | 解决方案、项目说明 |
| 工作手册（团队共享） | 知识库：工作手册 | 项目模板、流程规范 |
| 公司制度 | 知识库：公司制度 | 规章制度 |

### 笔记的生命周期

```
新想法/会议记录
  → 先写在 Obsidian 日记中（0-Daily/）
  → 整理后归入 PARA 对应位置
  → 如果需要团队共享 → 同步到飞书知识库
  → 项目结束 → 归档到 4-Archives/Projects/
```

### Obsidian 独有（不同步到飞书）

- 个人日记（`0-Daily/`）
- 个人项目（`1-Projects/Personal/`）
- 个人生活领域（`2-Areas/Personal/`）
- 学习笔记、读书笔记
- 迁移归档（`4-Archives/Notes/`）

## 二、任务管理

### 任务系统架构

```
飞书任务（权威数据源）
  ↕ 双向同步（lark-cli）
Obsidian 任务（本地视图）
  → 日记看板（每日任务聚合）
  → 项目任务清单（1-任务.md）
  → 综合任务清单（飞书任务/*.md）
```

### 飞书任务清单结构

飞书端按两类组织任务清单：

**项目任务** — 每个进行中项目一个清单：
- AI工作流、平台更新v2.7、广州机场、上港仓储管理
- 新太项目、南宁机场项目、PN仓库项目
- 洛阳化工厂、麦钉项目、测试流程建立

**综合任务** — 持续性职责领域：
- 需求管理、服务器运维、售前工作
- 营销宣传、财务工作、设备测试

### Obsidian 任务文件映射

| 飞书清单类型 | Obsidian 文件 | 说明 |
|------------|-------------|------|
| 项目任务 | `1-Projects/Work/项目名/1-任务.md` | 每个项目独立文件 |
| 综合任务 | `2-Areas/Work/飞书任务/清单名.md` | 统一放在飞书任务目录 |

### 任务操作流程

**新建任务：**
1. 在 Obsidian 对应文件中写入任务
2. 自动同步到飞书（创建飞书任务，回填链接）
3. 或在飞书端创建，下次拉取同步到本地

**完成任务：**
1. 在任一端标记完成
2. 同步到另一端

**每日任务查看：**
- 打开当天日记 → Tasks 插件自动聚合所有到期/进行中任务
- 或打开 `任务总览.md` 查看全局看板

### 任务同步规则

详见 [[飞书任务同步方案]]（steering: task-sync.md）

关键规则：
- 飞书是权威数据源，冲突时以飞书为准
- 项目任务必须有开始日期和截止日期
- 综合任务时间可选
- 同步范围：未完成 + 近 30 天已完成

## 三、文档协作

### 场景一：我写的文档需要团队看

```
1. 在 Obsidian 中写好 Markdown
2. 用 lark-cli 创建飞书文档，粘贴内容
3. 放入对应知识库节点
4. Obsidian 中标注飞书链接（frontmatter: feishu_url）
```

### 场景二：飞书上的协作文档

```
1. 在飞书上直接编辑（多人协作）
2. 定期拉取到 Obsidian 归档（docs +fetch）
3. 本地保留 Markdown 副本作为长期存储
```

### 场景三：会议纪要

```
1. 飞书自动生成智能纪要
2. 拉取到 Obsidian（4-Archives/Notes/Feishu/）
3. 提取关键信息到日记或项目文档
```

## 四、目录映射总览

### Obsidian → 飞书知识库

| Obsidian | 飞书知识库 | 同步方向 |
|---------|----------|---------|
| `1-Projects/Work/` | 进行中项目 | 双向（任务同步 + 文档共享） |
| `4-Archives/Projects/Work/` | 已归档项目 | 单向（归档时推送） |
| `3-Resources/Tech/` | 技术文档 | 按需推送（团队需要时） |
| `3-Resources/Business/` | 业务资料 | 按需推送 |
| `2-Areas/Work/`（流程/模板） | 工作手册 | 按需推送 |
| `2-Areas/Work/公司制度/` | 公司制度 | 按需推送 |

### 不同步到飞书

| Obsidian | 原因 |
|---------|------|
| `0-Daily/` | 个人日记 |
| `1-Projects/Personal/` | 个人项目 |
| `2-Areas/Personal/` | 个人生活 |
| `3-Resources/Knowledge/` | 个人知识 |
| `4-Archives/Notes/` | 迁移归档 |

## 五、日常工作流

### 早上开工

1. 打开 Obsidian 日记 → 查看今日任务（自动聚合）
2. 打开飞书 → 查看消息和会议安排
3. 如果有新飞书任务 → 同步到 Obsidian

### 工作中

- 写技术方案/项目文档 → Obsidian
- 需要同事协作编辑 → 飞书文档
- 新任务 → 在 Obsidian 创建，自动同步飞书
- 完成任务 → 在任一端标记，同步另一端
- 临时笔记/想法 → Obsidian 日记

### 下班前

1. 检查今日任务完成情况
2. 在日记中记录当天进展
3. 如有需要，同步任务状态

### 周末/月末

- 周回顾：检查各项目进度，更新任务优先级
- 月回顾：归档已完结项目，整理 Resources 中的笔记
