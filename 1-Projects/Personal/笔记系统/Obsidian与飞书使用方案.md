# Obsidian 与飞书使用方案

## 定位分工

| 维度 | Obsidian | 飞书 |
|------|---------|------|
| 定位 | 个人知识管理中心 | 团队协作平台 |
| 笔记 | 所有笔记的主库（PARA） | 团队共享文档（知识库） |
| 任务 | 本地视图 + 日记看板 | 权威数据源 + 团队协同 |
| 文档 | Markdown 长期存储 | 在线协作编辑 |
| 同步 | 飞书 ↔ Obsidian 双向同步（lark-cli） | — |

核心原则：**飞书负责团队协作，Obsidian 负责个人沉淀**。

## 一、目录架构

### 判断标准

| 层级 | 判断 | 放什么 |
|------|------|--------|
| Projects | 有终点吗？做完就归档 | 客户项目、产品迭代、内部改进 |
| Areas | 持续负责吗？有任务跟踪 | 按公司职能分，随业务演进 |
| Resources | 做事时查吗？定下来不变 | 技术参考、商业资料 |
| Archives | 不活跃了？冷藏 | P/A 的镜像归档 |

Areas vs Resources 的细分标准：**谁在驱动它改？**
- 随产品/业务演进会改的 → Areas（正在管）
- 决定了就不变的 → Resources（查阅用）

### Areas vs Resources 边界示例

同一主题的内容分两处，按"管/查"区分：

| 主题 | Areas（正在管） | Resources（查阅用） |
|------|---------------|-------------------|
| 供应商 | `设备管理/凡星位航UWB/对接记录.md` | `Business/供应商/凡星位航/产品手册.pdf` |
| 产品架构 | `产品研发/系统架构图.md`（随版本更新） | `Tech/代码片段/私有化部署手册.md`（通用不变） |
| 运维 | `运维管理/巡检清单.md`（持续维护） | `Tech/环境配置/集群参数.md`（查地址用） |
| 售前 | `业务管理/客户A/对接记录.md`（正在跟） | `Business/售前SOP/报价流程.md`（通用模板） |
| 展会 | `品牌宣传/2026深圳展会准备.md`（正在筹备） | `Business/PPT模板/`（取用素材） |

### 归档规则

不活跃的内容从 P/A 移到 Archives 对应位置：

| 场景 | 操作 |
|------|------|
| 项目完结 | `1-Projects/Work/X/` → `4-Archives/Projects/Work/X/` |
| 售前客户没谈成 | `2-Areas/Work/业务管理/X/` → `4-Archives/Areas/Work/业务管理/X/` |
| 供应商不再合作 | `2-Areas/Work/设备管理/X/` → `4-Archives/Areas/Work/设备管理/X/` |
| 展会结束 | 展会文档移到 `4-Archives/Areas/Work/品牌宣传/` |

频率：每季度扫一次，超过 3 个月没动的考虑归档。

## 二、任务管理

### 任务系统架构

```
飞书任务（权威数据源）
  ↕ 双向同步（lark-cli）
Obsidian 任务（本地视图）

  → 项目任务清单（1-任务.md）
```

### 任务操作流程

**新建任务：**
1. 在 Obsidian 对应文件中写入任务
2. 自动同步到飞书（创建飞书任务，回填链接）
3. 或在飞书端创建，下次拉取同步到本地

**完成任务：**
- 在任一端标记完成，同步到另一端

**每日查看：**
- 打开当天日记 → Tasks 插件自动聚合所有到期/进行中任务

### 任务同步规则

详见 steering: task-sync.md

关键规则：
- 飞书是权威数据源，冲突时以飞书为准
- 项目任务必须有开始日期和截止日期
- 综合任务时间可选
- 同步范围：未完成 + 近 30 天已完成

## 三、文档协作

### 场景一：我写的文档需要团队看

```
1. 在 Obsidian 中写好 Markdown
2. 用 lark-cli 创建飞书文档，粘贴内容
3. 放入对应知识库节点
4. Obsidian 中标注飞书链接（frontmatter: feishu_url）
```

### 场景二：飞书上的协作文档

```
1. 在飞书上直接编辑（多人协作）
2. 定期拉取到 Obsidian 归档（docs +fetch）
3. 本地保留 Markdown 副本作为长期存储
```

### 场景三：会议纪要

飞书自动生成的智能纪要默认在个人文档根目录下，处理流程：

```
飞书根目录（未处理）
  → 阅读纪要，提炼关键信息到 Obsidian 当天日记
  → 飞书端按会议内容移动到对应项目/知识库目录
  → 移动完成 = 已提炼处理
  → 根目录下不再有该纪要
```

判断分类目标：
- 项目相关会议 → 移到对应项目知识库节点
- 技术讨论 → 移到技术文档知识库
- 综合/管理会议 → 移到工作手册/会议记录

不拉取到 Obsidian 归档（量大且价值密度低，关键信息已提炼到日记）。

## 四、Obsidian ↔ 飞书知识库映射

### 同步到飞书的内容

| Obsidian | 飞书知识库 | 同步方向 |

| `1-Projects/Work/` | 进行中项目 | 双向（任务同步 + 文档共享） |

| `3-Resources/Tech/` | 技术文档 | 按需推送 |
| `3-Resources/Business/` | 业务资料 | 按需推送 |
| `2-Areas/Work/`（流程/规范） | 工作手册 | 按需推送 |

### 不同步到飞书

| Obsidian | 原因 |
|---------|------|
| `0-Daily/` | 个人日记 |
| `1-Projects/Personal/` | 个人项目 |
| `2-Areas/Personal/` | 个人生活 |
| `3-Resources/Knowledge/` | 个人知识 |
| `4-Archives/Notes/` | 迁移归档 |

## 五、日常工作流

### 早上开工

1. 打开 Obsidian 日记 → 查看今日任务（自动聚合）
2. 打开飞书 → 查看消息和会议安排
3. 如果有新飞书任务 → 同步到 Obsidian

### 工作中

- 写技术方案/项目文档 → Obsidian
- 需要同事协作编辑 → 飞书文档
- 新任务 → 在 Obsidian 创建，自动同步飞书
- 完成任务 → 在任一端标记，同步另一端
- 临时笔记/想法 → Obsidian 日记

### 下班前

1. 检查今日任务完成情况
2. 在日记中记录当天进展

### 周末/月末

- 周回顾：检查各项目进度，更新任务优先级
- 月回顾：归档已完结项目，整理 Resources
- 季度清理：扫 Areas 下不活跃的内容，考虑归档
