### 工作进展

#### 上港仓储 App 优化

1. 快速重新接任务（6440157）
   - MainFragment 的 onReaderEventListener 和 onRfidEventListener 中，新增 lastAddress 记录上次触发任务的标签地址
   - 同一货物掉落后重新叉起时，等待时间从 1000ms 缩短到 500ms，对话框自动确认从 2s 缩短到 100ms，几乎立即恢复任务

2. 修复窗口不隐藏（137883e）
   - MainFragment 中补充了任务卡片隐藏逻辑的遗漏

3. RFID 标签检测模式动态调整（c243507）
   - ReaderManager 和 RfidManager：新增 currentRfid、currentEnterMode、currentExitMode、defaultEnterMode、defaultExitMode 五个属性
   - 新增 HARD 阈值常量（enter RSSI -40、enter count 1、enter cache 3、exit RSSI -60）
   - TagTask.run 中通过 rfid == currentRfid 区分目标标签和其他标签，分别应用 current/default 阈值
   - 原有的固定阈值判定改为根据 mode 动态选择宽松/严格阈值
   - MainFragment：新增 setRfidModeForTask() / resetRfidMode() 辅助方法
   - startTask 成功时：目标标签 enter=1 exit=1，defaultEnter=1（其他标签严格进入防误扫）
   - predictShelfResult observer 中：predictShelf == droppingShelf 时目标标签 currentExitMode→0（放宽离开，方便触发完成）
   - finishTask 回调中无论成功失败都 resetRfidMode（因为 ViewModel 一进入就清空 task）
   - stopTask、读卡器重连回调中 resetRfidMode

4. 业务流程文档（9bce65c）
   - 新增 [[1-Projects/Work/上港仓储管理/仓储出入库业务流程]]，涵盖业务概述、货物状态机、系统架构、出入库六阶段流程、NearbyShelfManager 货位预测机制、RFID 检测机制与动态调整策略、状态流转图、代码调用位置

5. 完成任务对话框区分入库/出库（7c95b5e）
   - showTaskFinishDialog 拆分为：
     - showCheckoutTaskFinishDialog：出库场景，简单确认对话框，shelf=null，2s 自动确认
     - showCheckinTaskFinishDialog：入库/移库场景，单选列表对话框，候选库位按优先级排列（精确命中 > droppingShelf > predictShelf > nearbyShelf，去重），推荐项标记 (推荐)
   - 精确命中（predictShelf == droppingShelf）时 2s 自动确认，不匹配时 5s 给操作员更多时间选择
