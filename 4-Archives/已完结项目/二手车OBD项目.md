# 二手车 OBD 项目

## 设备移除告警需求

1. 提交销售申请时，不允许处于设备移除告警的车辆提交销售申请
2. 车辆被售出时，把车辆的设备移除告警结束掉

## 车辆绑定异常问题

原因：
1. 车辆打火之后才能确定设备与当前车辆是否为绑定关系
2. 部分数据是在车辆打火之前传上来的

解决方案：
1. 设备插入事件不带 vinCode 信息，不能作为消除设备移除告警的依据，需在打火之后才消除
2. 设备移除告警消除之前，收到的数据（位置数据和告警触发信号）不做任何处理

特殊情况：OBD 设备没电时，拔除设备无法触发设备移除告警，需重新插入设备时才能触发。

# 2022 年详细记录

## 车辆状态定义

### 车辆状态
1. 已录入 CREATED：录入系统后默认状态
2. 已绑定 BOUND：绑定设备后状态
3. 已出售 SOLD：出售后状态，会同时把设备设置成 REMOVABLE
4. 已解绑 UNBOUND：解绑后的状态

### 设备状态
1. 创建 CREATED：录入系统后默认状态
2. 激活 ACTIVATED：激活系统后状态
3. 已绑定 BOUND：绑定接口后
4. 可解绑 REMOVABLE：已过户/申请绑定后设置
5. 已过期 EXPIRED：设备过期后

## 车辆监管状态

| 状态 | 说明 |
|------|------|
| 正常监管 | 当前没有任何激活告警，排除托管车辆 |
| 违规出圈 | 当前触发围栏告警 |
| 低电压 | 当前触发低电压告警 |
| 脱离监管 | 当前触发移除设备告警 |
| 临时托管 | 当前处于托管状态 |

备注：临时托管状态下的车辆不触发任何告警（待确认）

## 设备接口需求

- @LB 新增接口：设备增加有效期字段，设备到期延长激活时间接口
- 绑定车辆接口流程：线下插入设备 → 拿到设备和 VIN → 获取车辆列表查找 VIN → 绑定
- 设备操作记录提供绑定失败记录
- 不点火情况下不触发告警
- 12 小时没有绑定告警
- 设备 serialNum：根据经销商获取过期日期，设备列表加上日期，临期到期筛选

## 数据库视图方案

关于数据库扩展表方式无法提供足够数据，改用视图方式：每个接口的对象都会生成一个视图，视图字段与接口返回字段对应，同时提供两类设备的原始数据。
