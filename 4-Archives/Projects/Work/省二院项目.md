# 省二院项目

医院定位项目，主要给保安和保洁员做定位管理平台。

## 项目介绍

- 医院项目，给保安和保洁员做定位
- 基于之前叉车管理平台改造
- 1-2个月交付周期

## 开发计划

1. 跑通之前做的功能
2. 添加保安的增删改查
3. 添加保洁员的增删改查
4. 添加新的巡检模块
5. 添加新的考勤模块
6. 把之前UC里的数据分析移植过来
7. 先写mock接口，功能对接好后再对接真正后端
8. 通用功能模块化（绑定、搜索、列表等）

## 巡检功能需求

### 巡检周期任务
- 巡检点（地图）/ 点半径或多边形 / 顺序
- 巡检时间段/周期
- 作用的人或组，是否可以重复
- 是否启用
- 软删除，保留每次修改记录
- 排除假期

### 巡检子任务
- 巡检任务开始时，自动为相关人员创建子任务
- 经过所有巡检点或过了巡检时间结束任务
- 记录经过和未经过的巡检点个数，标记是否完成
- 巡检任务人员经过巡检点的记录

## 考勤功能需求

### 考勤任务
- 考勤点 / 点半径或多边形

# 智慧医院位置管理服务平台 V1.0（Yuque 迁移）

## 项目需求

现代医疗专业化程度越来越高，医院对人员管理需求很高。本次智慧医院位置管理服务平台连接互联网与智慧医疗管理系统硬件，实现省二院被定位人员的工作规范化、高效化。

## 平台角色设计

| 角色 | 业务需求 |
|------|----------|
| 系统管理员 | 独立账户，支持部门/角色增删改查，查看所有数据分析和预警信息 |
| 保洁管理员 | 独立账户，保洁员信息管理（支持批量导入），工作区域围栏/时间划分，部门数据分析 |
| 保安管理员 | 独立账户，保安员信息管理（支持批量导入），工作区域围栏/时间划分，部门数据分析 |

## 平台功能模块

| 功能模块 | 功能概述 |
|----------|----------|
| 人员管理 | 编辑人员信息，查看实时位置和状态（空闲/忙碌），搜索定位，紧急调度 |
| 运动轨迹 | 室内外工作人员位置信息，AOA 基站精确度≤1米，蓝牙基站以基站连线形式呈现 |
| 轨迹回放 | 记录移动轨迹，支持单人回放，支持同时查看 5 个定位对象轨迹 |
| 考勤 | 进入考勤区域自动上班打卡，离开自动下班打卡，显示在岗/缺岗人数 |
| 巡更 | 设置起点/终点路径，进入路径区域视为上班打卡，可设定每天巡更更次 |
| 执法记录仪联动 | 实时定位、实时视频传输，双向语音对讲，4G 网络 |
| 电子围栏管理 | 可定制化围栏设定，离开工作区域告警，闯入非工作区域告警 |
| 告警管理 | 围栏告警、异常停留告警、不在岗告警、离线告警、低电量告警；支持页面气泡/icon/系统通知/全局提示 |
| 数据统计 | 围栏数据、告警次数统计、历史轨迹数据统计 |
| 三维地图与模型 | 室外/室内模型全方位展示，POI 信息（名称/经纬度/楼层/属性） |
| 大屏展示 | 实时数据查看，楼宇/楼层切换，定位对象工作状态查询 |
| 监控视频联动 | 预留接口，摄像头管理，录像管理 |
