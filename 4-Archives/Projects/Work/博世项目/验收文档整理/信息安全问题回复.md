# 信息安全问题回复

## 边缘侧

数据隐私保护：需要获取手机蓝牙的标识，微信 ID，用户的位置数据 ，边缘服务器内部只传输原始的蓝牙数据，不解算，不储存任何和位置相关的数据，所有的解算在定位服务器中完成。

莱讯：
网络安全 : 蓝牙广播,手机操作系统会对 mac 地址做随机化操作，所以广播出来的数据第三方无法匹配到对应手机。

物理（硬件）安全

应用安全 – 开放端口22 ，ssh 登录管理 ， linux 账号体系

边缘设备系统应用的保护措施
Hardening server:
- Remove or Disable Unnecessary Services, Applications or Network Protocols
- Reduce the Number of Open Network Ports
- Forbid root logins from the network if possible.
- Run as less processes as root as possible. Use the least privilege principle for all processes.
- Security event log records must be kept for at least 6 months
- If possible, only privileged users should have access to USB media

传输加密： TLS1.2+ shall be used for cloud communication ; 边缘计算器在通过 poe 网线收到数据以后，通过 加密MQTTS（MQTT with TLS） 将数据发送到地图服务器

莱讯回复：目前的方案定位引擎服务不部署在边缘侧，原始数据直接通过UDP协议传到了云端进行解算，针对服务器应用的安全，我们已经做了一系列应用安全措施，包括移除或禁用不必要的服务、减少开放的网络端口数量，禁止root 用户的登陆、以及强化Linux账号体系，以降低服务器遭受攻击的风险。


## 云端

DSO：对重要数据和个人信息等数据
- 访问控制 - 见账号多租户系统说明.docx
- 审计机制 – 系统日志有记录
- 监测措施 – 一年
- 备份机制 - 定期备份
- 存储机制
  - 存储时间一般为一周到一个月。
  - 一般是储存轨迹数据和用户的微信名称数据库中储存，数据库中做模糊处理，不明文储存，脱敏存储
  - 定位服务器也不储存任何的定位结果，定位结果解算完成以后，直接推送。

莱讯回复：
- 审计机制：云端服务器存在日志记录审计机制，以便监视系统活动并生成相应的审计日志，以追踪系统使用情况、识别安全问题等，日志储存时间保存时间为一个月。
- 检测措施：服务器提供了CPU、内存、磁盘、网络监控功能，通过监控页面可以实时了解各个服务器节点和各个服务的运行状态，当发生资源使用异常情况，会通过邮件的方式通知运维人员，以确保系统的稳健性和安全性。
- 备份机制：
  - 关键服务器会每天进行全磁盘镜像备份
  - 数据库会每天进行增量备份，每7天进行全量备份
  - 对象存储采用多数据节点冗余存储，且在发生错误后能自动恢复

## 网络安全

莱讯回复：
主要使用了腾讯云提供的安全组功能来控制服务器对外暴露的端口，只对外暴露必要的端口供外部使用。同时如果通过监控发现大量非法流量，能快速开启Web应用防火墙(WAF)以防止对服务器造成进一步破坏。集群服务器实施访问控制，仅授权需要访问服务器的人员，并使用最小权限原则，限制用户的权限。

整个SAAS系统均采用了集群高可用设计，且在部署时候都考虑了冗余部署。也即每个应用会均匀分布在多个不同的物理节点，再由多个物理节点组成SAAS服务的系统集群。这样能做到少量的应用或节点宕机后，不影响到整个系统的可用性。

- 身份认证和授权： 使用 Kubernetes 内建的 RBAC（Role-Based Access Control）进行身份认证和授权，以确保只有授权的用户可以访问集群资源。
- 配置网络策略以限制 Pod 之间和 Pod 与集群外部的流量。使用 Network Policies 来定义允许和拒绝的通信规则。
- 使用 Pod 安全策略（Pod Security Policies）限制容器的权限，例如限制使用特权容器、文件系统访问等。
- 敏感信息管理： 使用 Kubernetes 的 Secrets 和 ConfigMaps 来安全地管理敏感信息，如密码、API 密钥等。 避免在容器中直接硬编码敏感信息。
- 审计和日志：启用审计功能以记录集群中的活动，并定期审查日志以检测异常行为。集中管理和分析日志，以更好地理解集群的安全状态。
- 更新和漏洞管理： 定期更新 Kubernetes 集群的版本，以获取最新的安全补丁。 使用容器镜像仓库中的最新基础镜像，并定期扫描镜像以检测漏洞。
- 所有浏览器/三方服务器到SAAS服务的HTTP/WebSocket流量均采用SSL/TLS加密，存储数据会使用端到端加密存储落盘。

## 数据安全

传输链路加密
- Passwords and other authentication secrets must be cryptographically protected during transport. Do not use plain text credentials in the payload of an HTTPS request
- Security event log records must be kept for at least 6 months

Password policy：
- at least 12 characters long password
- has to contain at least one capital letter
- has to contain at least one low letter
- has to contain at least one special character
- has to contain at least one number
- doesn't contain the context specific words nor dictionary words
- doesn't contain the sequential and repetitive characters

莱讯回复：
启用服务器和应用程序的日志记录，以便及时检测异常行为。设置实时监控系统，以识别潜在的安全威胁。

## 其他

Cryptography：
- Minimal key length: RSA: 2048bit, AES: 128bit
- Hash algorithms: SHA2 or SHA3, at least SHA256. Don't use SHA1 any longer!

莱讯回复：
- Token的生成过程采用了业界领先的SHA-256算法进行加密处理，具有高强度的加密能力和卓越的抗碰撞性，确保Token的唯一性和安全性，有效防止数据被篡改或伪造。
- 用户密码存储，采用了业界广泛认可的BCrypt算法来存储密码，确保用户密码的安全性，抵御暴力破解和彩虹表攻击。在设置BCrypt算法时，我们选择了强度为12的配置，密码会经过12轮复杂的哈希运算，生成一个极其难以逆向破解的加密字符串。

## 应急管理措施

- 数据加密：使用加密技术保护存储在数据库、文件系统或传输过程中的敏感数据，以防止未经授权的访问。
- 访问控制：实施严格的访问控制措施，例如基于角色的访问控制（RBAC）或多因素身份验证（MFA），以确保只有授权用户可以访问敏感数据。
- 安全审计和监控：实施系统日志记录和监控措施，及时检测和响应可疑活动，以减少数据泄露的风险。
- 消息通知：一旦发现数据泄露，通过邮件的方式通知运维人员尽快处理。
