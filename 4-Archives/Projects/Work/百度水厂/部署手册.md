# 部署手册

## 服务器环境准备

### 1. 软件包准备

#### 1.1. 默认源替换

替换为清华源

```bash
sed -e 's|^mirrorlist=|#mirrorlist=|g' \
    -e 's|^#baseurl=http://mirror.centos.org|baseurl=https://mirrors.tuna.tsinghua.edu.cn|g' \
    -i.bak \
    /etc/yum.repos.d/CentOS-*.repo
```

更新缓存

```bash
yum makecache
```

#### 1.2. EPEL 扩展源

安装 EPEL

```bash
yum install epel-release
```

文本替换为清华源

```bash
sed -e 's!^metalink=!#metalink=!g' \
    -e 's!^#baseurl=!baseurl=!g' \
    -e 's!//download\.fedoraproject\.org/pub!//mirrors.tuna.tsinghua.edu.cn!g' \
    -e 's!//download\.example/pub!//mirrors.tuna.tsinghua.edu.cn!g' \
    -e 's!http://mirrors!https://mirrors!g' \
    -i /etc/yum.repos.d/epel*.repo
```

#### 1.3. 安装必要的软件

如果需要 nginx 也可以一并在此时安装

```bash
yum install -y bash-completion bridge-utils
```

#### 1.4. 全面更新一次软件包

```bash
yum update -y
```

### 2. 内核准备

修改内核配置和周边配置以适配 K8s 的安装

#### 2.1. 关闭防火墙

```bash
systemctl disable firewalld --now
```

#### 2.2. 修改内核参数

```bash
echo br_netfilter > /etc/modules-load.d/br_netfilter.conf
```

### 3. 磁盘准备

需要我们确定磁盘的初始分配并执行微调

#### 3.1. 查看磁盘分配

通过命令查看磁盘分配

```bash
df -hT
```

如果发现 根目录 / 分配容量过小 (通常是50G)，而 /home 目录分配的绝大部分磁盘，则需要做盘内分区调整。因为 K8s 的运行是不使用 /home目录的

#### 3.2. 盘内分区调整

备份并卸载 /home 目录

```bash
cp -r /home /tmp/
umount /home
```

降低 /home 目录的空间，其中 `<size>` 为需要降低的数目

```bash
lvreduce -L -<size>G /dev/mapper/centos-home
```

重新格式化

```bash
mkfs.xfs /dev/mapper/centos-home -f
```

重新挂载 /home 目录，如有需要，可从 /tmp 恢复备份

```bash
mount /dev/mapper/centos-home /home
```

扩容根目录 / ，其中 `<size>` 为之前释放出来的空间

```bash
lvextend -L +<size>G /dev/mapper/centos-root
```

使修改生效

```bash
xfs_growfs /dev/mapper/centos-root
```

## k3s集群搭建

### 1. 配置仓库

集群参数配置均可使用文本配置在YAML文件中：

```
.
├── .cert             // 集群认证相关
├── .gitlab-ci.yml    // Gitlab CI相关配置
├── README.md
├── business          // 业务命名空间下的配置
├── cluster           // 集群配置
├── oam               // 运维命名空间下的配置
├── public            // 网络接入层命名空间下的配置
└── system            // K8s系统命名空间下的配置
```

#### 1.1. 准备 K3s 配置

集群配置结构如下：

```
cluster 
├── master 
│   ├── config.yaml       // Master节点配置
│   └── registries.yaml   // Master节点Registry相关配置
└── worker
    ├── config.yaml       // Worker节点配置
    └── registries.yaml   // Worker节点Registry相关配置
```

##### 1.1.1. Master节点配置

一个典型的 config.yaml 内容如下：

```yaml
# logging
log: "/var/log/k3s/k3s.log"

# database
datastore-endpoint: "Sqlite"

# client
write-kubeconfig-mode: "0644"

# agent
node-label:
  - "cpu=2"
  - "memory=4"
  - "variant=online"
  - "role=master"
  - "cluster=k3s"
private-registry: "/etc/rancher/k3s/registries.yaml"

# components
disable:
  - "traefik"
```

其中，需要注意的是：配置中把 traefik 禁用了，是因为我们自己搭建了网络接入口层，因此不需要使用 K3s 内置的 traefik 组件
