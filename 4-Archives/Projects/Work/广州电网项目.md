# UWB-map 服务器部署流程（Yuque 迁移）

## 1. 安装虚拟机

- 安装 VMware Pro
- 解压缩 map 虚拟机文件到文件夹
- 从文件夹启动

## 2. 配置服务器静态 IP

```bash
vi /etc/sysconfig/network-scripts/ifcfg-ens33

BOOTPROTO="static"
IPADDR=192.168.1.6
GATEWAY=192.168.1.254
NETMASK=255.255.255.0

service network restart
```

## 3. 查看 UWB-RTLS 服务器配置

- 获取 UWB-RTLS 的 IP 地址、登录账号密码
- 在 SitePlan 里找到原点，确认参考点的相对位置
- 查看项目 ID 和标签定位对象 ID（obj_uid）

## 4. 修改 UWB-map 服务器配置

```bash
cd ~/deploy
vi docker/.env

# 环境变量说明
REDPOINT_EMAIL=xxx@123.com        # 从 UWB 系统获取
REDPOINT_LATITUDE=24.805214       # UWB 系统原点纬度
REDPOINT_LONGITUDE=113.581062     # UWB 系统原点经度
REDPOINT_PASSWORD=[password]      # 从 UWB 系统获取
REDPOINT_PROJECT=xxx              # 从 UWB 系统获取
REDPOINT_SERVER=http://192.168.x.x  # UWB 系统地址
MAP_SERVER=http://192.168.x.x:8282  # 本系统地址 + 8282 端口
MAP_APP_ID=wmsnavigation
MAP_APP_SECRET=bc608e36-7d60-44fa-ad6e-6bbee33b06b6
MAP_SPACE_ID=x                    # 根据地方修改
MAP_ID=x                          # 根据地方修改
MAP_FLOOR=1
NOTIFICATION_WAIT_TIME=100

./script.sh start
```

## 5. 更新地图

- 进入 pgAdmin `http://192.168.x.x:8080`
- 复制 wmsnavigation.sql 内容并导入

## 6. 更新后端程序

```bash
cd ~/deploy
./script.sh down
docker rmi reliablesense.tencentcloudcr.com/wms-project/backend:dev
docker load < backend.tar
./script.sh start
```

## 7. UWB 标签设置

```bash
CFG UART cmd=on       # 通过串口打开标签，波特率 115200
CFG BLE m=COM          # 设置蓝牙模式
CFG OUT en=A,P,M       # 打开输出
CFG IMU m=off          # 关闭 IMU
```

# 项目问题汇总（Yuque 迁移）

## Map 服务问题

- 不明错误导致接口返回 403 Forbidden，跟授权登录有关，可能是项目 ID 输入错误
- 服务器重启后 WCS 可能断开 MQTT 连接，需要重新启动 WCS 软件
- 没有有效日志，导致现场问题不好排查
- Docker 没有响应，程序没起来

## RTLS 服务问题

- 快照恢复后时序可能不正确，导致定位不准，需要重新调整时序
- 从离线文件恢复后电子围栏数据异常
- 从离线文件恢复后项目 ID 和标签 ID 有变化，WCS 也需要修改
- 断电或非正常关机后导致数据库无法启动
- 标签重启后 IMU 从 off 恢复成 af
- 电子围栏需要备份导出导入

## 终端程序问题

- 移库流程有 Bug（已修复：茂名、阳江、云浮、湛江）
- 终端 IP 需要可配置
- 会遇到下发的指令没法查出终点起点的情况（射频门名字没对上）

## 网络问题

- IP 地址被占用，导致红点服务器不能正常访问

## 虚拟机问题

- 断电或非正常关机后导致系统损坏
- 存放虚拟机的文件夹离奇失踪
- 需要虚拟机操作手册指导现场人员
