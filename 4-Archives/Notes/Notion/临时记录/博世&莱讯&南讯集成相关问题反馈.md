# 博世&莱讯&南讯集成相关问题反馈

一、基础鉴权部分
1、目前仅看到了登入/登出，无相关鉴权接口。辛苦帮忙解答所有接口是否需要基础鉴权。

接口都需要进行鉴权，有两种方式

1. 登录会返回 token 字段，后续通过传入 X-AUTH-TOKEN  和 X-Schema 头，访问其他接口
2. 通过传入项目特定的 apiKey 和 apiSecret 头来访问其他接口

二、地图基础数据部分
1、获取地图列表/获取地图详情接口，里面如何获取标准的室内地图 + 地图基础坐标系数据 ？

答：室内地图的数据由 多边形元素，兴趣点，模型等元素进行渲染，可以通过以下接口获取相关地图元素

1. 地图多边形  /api/maps/{mapId}/polygons （接口文档：/地图服务相关接口/地图元素相关接口/获取地图元素的地理信息）
2. 地图兴趣点  /api/maps/{mapId}/pois （接口文档位置：/地图服务相关接口/兴趣点相关接口/获取兴趣点的地理信息）
3. 地图模型  /api/maps/{mapId}/scenegraphs （接口文档位置：/地图服务相关接口/地图实物模型相关接口/获取地图实物模型的地理信息）

2、地图基础数据发生变更后，通过何种方式处理？

注：因地图上会设置相关自定义活动点，此部分基础数据需同步至南讯侧，才能开展相关业务。无法实时调用莱讯接口直接开展业务。

答：目前如果需要修改地图兴趣点，可以登录管理后台进行在线编辑。若其他地图基础数据发生变更，可以把相关地图变化告知我们，我们进行具体地图元素的更新，元素更新后，对应接口会返回更新后的地图数据。

三、实时定位获取
1、在《定位平台小程序SDK》中，初始化方法是会传入用户openID，关于openid &  coreaiot_generate_service_uuids方法内转码过的ID 的映射关系是会在莱讯系统里面存储一份吗 ？

答：当通过小程序SDK，调用 initLbs 后，会完成该小程序用户跟我们平台的建档，在我们系统里会储存一份openId 与 转码后Id 的映射关系。

2、小程序端用户的实时位置，可直接通过SDK内（ 可传入回调函数接收位置更新响应）来获取用户的实时位置吗？

答：小程序SDK可以通过调用 startLocation 开始进行定位，可以传入回调函数获取用户最新位置，同时地图里会自动显示用户实时位置。

四、用户轨迹数据获取
1、未看到相关中间件订阅模式或相关接口能按时间段同步相关用户轨迹信息，需要帮忙解答一下此部分的内容。

答：位置数据的可以通过两种方式获取

1. 通过订阅位置信息Websocket接口 /websocket/position/api/ws，实时接收最新的位置推送 （接口文档位置：/位置服务相关接口/定位数据相关接口/位置信息Websocket推送）
2. 通过 调用 /api/positions 接口，可以传入特定时间段，获取用户在特定时间的轨迹数据 （接口文档位置：/位置服务相关接口/定位数据相关接口/获取位置数据）
