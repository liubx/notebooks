# 后端修改建议

- [ ]  修改围栏人数统计接口 /fences/access 修改成 /fences/statistics/accesses
- [ ]  添加围栏进出记录分页接口 /fences/accesses
    - 请求字段：
        
        ```bash
        {
        		current,
        		pageSize,
        		trackableIds,
        		trackableTypeIds,
        		typeIds,
        		fenceIds,
        		startTime = '2020-02-01T15:37:11.758386Z',
        		endTime = '2020-02-10T15:37:11.758386Z',
        		sort = 'id',
        		order = 'ascend',
        }
        ```
        
    - 返回字段
        
        ```bash
        {
          trackableId?: string;
          trackable?: TrackableData;
          fenceId?: string;
          fence?: FenceData;
          enterTime?: string;
          leaveTime?: string;
        }
        ```
        
- [ ]  围栏上下限告警的触发，当围栏超员或缺员时触发，没有超员或缺员时告警结束，记录relatedId为fenceId，记录 trackableId 为触发超额时的trackableId，或者不记录也可以，告警的位置可以记录围栏的位置，把这类告警区分成 超额告警 和 缺额告警
- [ ]  告警 title 和 description 统一整理
    - 标题 title （这个是不是改数据库的记录就可以）：
        
        ```bash
        1	违规进入 （现有的围栏告警如果不好分开，就直接归类到这个告警里）
        2 违规离开
        3	超额
        4 缺额
        5	滞留超时
        6	暂离超时
        
        11 低电量
        12 设备移除
        13 信号丢失
        
        ```
        
    - 描述 description，大括号内的文案
        
        ```bash
        违规进入：xxx 对象 {违规进入了 xxx围栏}, xxx 时间
        违规离开：xxx 对象 {违规离开了 xxx围栏}, xxx 时间 
        超额：{xxx 围栏超额}, xxx 时间 
        缺额：{xxx 围栏缺额}, xxx 时间
        滞留超时：xxx 对象 {停留 xxx围栏 超时}, 在 xxx 时间
        暂离超时：xxx 对象 {离开 xxx围栏 超时}, 在 xxx 时间
        
        低电量：xxx 对象 {绑定的设备处于低电量状态}, 在 xxx 时间  
        设备移除：xxx 对象 {绑定的设备被非法移除设备}, 在 xxx 时间 
        信号丢失：xxx 对象 {绑定的设备信号丢失}, 在 xxx 时间 
        ```
